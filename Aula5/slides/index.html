Adicionando um novo arquivo

  • Qualquer arquivo que não apresenta um Front-Matter, é considerado um arquivo regular.
  • Sendo assim, ele poderá ser acessado diretamente pela url correspondente ao local na árvore de diretórios
  • Caso queira ignorar algum arquivo ou pasta, declare-o em _config.yml
  • Acesse o exemplo jekyll_files para um exemplo completo

Geração de conteúdo com dados

  • A última forma de adicionar conteúdos ao Jekyll é através de arquivos de dados
  • Esses arquivos ficam em uma pasta especial chamada _data
  • Crie qualquer arquivo yaml que ele será reconhecido como dados
  • Para acessar os dados, basta referenciar:
    • 
{% for i in site.data.dados.outros_exemplos %}
  {% if i.idade < 18 %}
  - {{ i.nome }} (menor)
  {% else %}
  - {{ i.nome }} 
  - {{ i.idade }} 
  {% endif %}
{% endfor %}
